# Тестовое задание на позицию fullstack

## Задача

Создать список из 1 000 000 элементов (ID от 1 до 1 000 000) с интерфейсом для выбора и сортировки.

## Реализованный интерфейс

Интерфейс состоит из двух окон (контейнеров):

- Левое окно отображает все элементы, кроме выбранных.
- Правое окно отображает выбранные элементы.

Функции левого окна:

- Фильтрация элементов по ID.
- Инфинити-скролл: каждый запрос, в том числе при поиске, возвращает не более 20 элементов; следующие элементы загружаются порциями по 20.
- Добавление нового ID вручную. ID может быть любой непустой строкой.

Функции правого окна:

- Фильтрация элементов по ID.
- Сортировка выбранных элементов с помощью Drag&Drop, в том числе при включённой фильтрации.
- Инфинити-скролл: размер страницы не более 20 элементов.
- После обновления страницы сохраняются выбранные элементы и их порядок. Поисковая строка не сохраняется.

## Хранение и очередь запросов

- Сервер на Express.js хранит состояние в памяти процесса: выбранные ID и созданные вручную ID.
- Учётные записи отсутствуют: состояние общее для всех посетителей, пока работает сервер. После перезапуска сервера состояние очищается.
- Повторяющиеся запросы на чтение в очереди объединяются. Один и тот же ID нельзя добавить повторно.
- Чтение, выбор, удаление и изменение порядка обрабатываются раз в секунду.
- Добавление новых ID обрабатывается пакетами раз в 10 секунд.
- Сервер не создаёт массив из миллиона объектов: исходные числовые ID вычисляются по требованию. В DOM отрисовывается только видимая часть загруженного списка.

## Запуск локально

```sh
npm install
npm run dev
```

Откройте `http://localhost:5173`.

Для запуска собранного приложения одним процессом:

```sh
npm run build
npm start
```

Для размещения подходит хостинг с долгоживущим Node.js-процессом, например Render. Serverless-платформы не подходят: состояние задания хранится в памяти процесса.
